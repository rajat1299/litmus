# Litmus Alpha Release Notes

Date: 2026-03-31

This alpha is the first repository state that proves the grounded local loop end-to-end with a packaged CLI, a runnable demo repo, replay artifacts, a local stdio MCP surface, watch mode, and a GitHub Action that publishes PR comments.

## Included In This Alpha

- `litmus verify`
- `litmus replay <seed>`
- `litmus watch`
- `litmus mcp` over local stdio
- GitHub Action execution and PR comment publication
- mined invariants from simple pytest-style request/response fixtures
- conservative scenario building from mined and suggested invariants
- differential replay with replay-trace artifacts
- property checks over confirmed property invariants
- deterministic ASGI execution harness
- semantic simulators for HTTP, `httpx`, `aiohttp`, SQLAlchemy async, and Redis async
- runnable payment demo in `examples/payment_service`

## Supported Launch Stack

- Python 3.11+
- FastAPI / Starlette-style ASGI apps
- `httpx`
- `aiohttp`
- SQLAlchemy async
- Redis async

## Release Validation

This alpha release path is grounded by repository checks, not just docs:

- `uv build --out-dir dist`
- install the built wheel into a fresh virtual environment
- run the packaged `litmus` CLI
- execute the broken payment demo
- replay `seed:1`
- fix the demo and rerun verification to green

## Demo Path

Use `examples/payment_service/` for the current alpha story.

The example intentionally starts broken so the first run demonstrates:

- one unchanged replay
- one breaking replay
- replay artifact generation
- a concrete `litmus replay seed:1` explanation

After applying the documented fix in the example app, the rerun demonstrates the green path.

## Known limitations

- The top-level `README.md` remains aspirational and intentionally describes a broader future CLI surface than this alpha currently ships.
- The MCP surface is local stdio only and returns the same local run/result model as the CLI.
- The demo proves the mined-baseline and replay loop, not the full aspirational DST/fault-injection story.
- Local mode still uses a lighter replay budget than CI mode.
- The GitHub Action publication path is functional, but API failures still surface through simple exception behavior rather than rich action diagnostics.
- Packaging validation is currently repository-driven; there is no automated publish-to-index workflow in this repo yet.

## Recommended Next Step

Treat this alpha as the first usable internal/demo release candidate. The next release-oriented pass should focus on publish automation, richer install docs, and narrowing the gap between the aspirational README surface and the grounded shipped surface.
