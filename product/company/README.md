# Litmus Company Blueprint

This folder captures the venture-scale company framing for Litmus.

The core thesis is:

> Litmus should become the confidence layer for AI-native engineering teams.

That means Litmus is not only a deterministic simulation tool, a PR bot, or an MCP server. It is the product that decides whether an AI-written change is trustworthy enough to ship, what evidence supports that decision, and what should happen next.

## Documents

- `category-definition.md`  
  Defines the market category, buyer, problem, and why Litmus can be venture-scale.
- `product-architecture.md`  
  Describes the system Litmus must become: change intelligence, verification engine, and confidence control plane.
- `0-to-1-wedge.md`  
  Defines the initial wedge, first users, first workflows, and the shortest path to a real paid product.
- `1-to-10-expansion.md`  
  Maps how Litmus expands from a reliability verifier into a broader AI engineering confidence platform.
- `moat-map.md`  
  Defines the defensibility layers that matter and the ones that do not.
- `gtm-and-pricing.md`  
  Lays out distribution, sales motion, packaging, and pricing.
